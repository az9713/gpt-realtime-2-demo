> i am not familiar with docker. what does it mean by " docker compose up -d postgres redis, then run migrations and seed. "                                                                                      
● Plain-language explanation, grounded in this project:

  Docker is a way to run pre-packaged software (a database, a web server, etc.) without installing each piece on your machine. Each "service" runs in its own isolated container, but they can talk to each other.                                                                                                                                                                                                                    Your docker-compose.yml defines five services:
  - postgres — the database that stores conversations, turns, tool calls, traces, approvals.
  - redis — fast in-memory store, used for pub/sub (cockpit ↔ core notifications).
  - core — your Python agent (FastAPI on port 8000).
  - edge — your Node transport (Fastify + WebSocket on port 8080).
  - frontend — the React cockpit (Vite on port 5173).

  Now what each command does:

  docker compose up -d postgres redis
  - up = start the listed services (downloads images first time, then runs them).
  - -d = "detached" — runs in the background so your terminal isn't tied up.
  - We start postgres and redis first because the agent core needs them up before it can connect.

  make migrate (which runs alembic upgrade head)
  - "Migration" = applying a recipe that creates the database tables. Right now Postgres is running but empty.
  - This runs the migration we wrote — core/alembic/versions/20260507_000001_initial.py — which creates the 5 tables: conversations, turns, tool_calls, approvals, trace_events.
  - Without this step, when Aria tries to log a conversation, Postgres would error out with "table doesn't exist."

  make seed-hvac (or bash scripts/seed-hvac.sh)
  - "Seed" = load realistic test data into the system.
  - This copies verticals/hvac/fixtures/*.json into data/hvac/ — that's the parts catalog, 12 trucks, 20 customers, 20 warranty records, and 30 scheduled jobs.
  - Without this, when Aria looks up a part the response would be empty.

  So the sequence is: start storage → create tables → load fixture data → start the rest of the stack. That's why we do them in that order rather than docker compose up everything at once.
